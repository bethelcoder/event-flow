Development Guide - Sprint 1

1. Project Overview

WHAT THE APP DOES: A full-stack real-time event management platform built with Express.js, MongoDB, EJS templating, and Socket.IO. Designed to help organizers manage event-day operations such as guest check-ins, live schedule updates, venue mapping, and on-site staff coordination.

Tech Stack:
Frontend: EJS, CSS, JS.
Backend: Node.js, Express Database MongoDB (Mongoose)

2. Setup Instructions

How to clone repo:

git clone https://github.com/bethelcoder/event-flow.git cd event-flow

Install dependencies:

npm install

.env variables:

MONGO_URI=
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=
GOOGLE_CALLBACK_URL=
JWT_SECRET=
SESSION_SECRET=

How to run the project:

npm run start

3. Development Database

Database Technology: MongoDB (with Mongoose ODM)

Steps to set up locally:

  1. Install MongoDB Community Edition or run it with Docker.
  2. Create a new local database called eventflow_dev.
  3. Update the MONGO_URI in your .env file to point to your localhost instance:
    MONGO_URI=mongodb://127.0.0.1:27017/eventflow_dev
  4. Start MongoDB so it runs in the background.

4. Development API

Backend Technology: Node.js with Express.js

Steps to run API locally:

  1. Navigate to the project root: cd event-flow
  2. Ensure dependencies are installed: npm install
  3. Start the server: npm run start
  4. The console should display: Server running on port 3000 and MongoDB connected.
  5. API endpoints are available under http://localhost:3000/api.

5. Development Site

Frontend Technology: EJS templates, CSS, and JavaScript served by Express.

Steps to run site locally:

  1. Ensure the API/backend server is already running with npm run start.
  2. Open http://localhost:3000 in your browser.
  3. The frontend (EJS views) will load from the same Express server and connect to the MongoDB database.
