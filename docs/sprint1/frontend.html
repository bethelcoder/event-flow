Frontend Documentation - Sprint 1

Overview

During Sprint 1, the frontend team focused on creating the user-facing pages of the Event Flow project. Our main goal was to establish a modern, clean, and responsive interface that serves as the foundation for user interactions. The pages are developed using EJS templates integrated with the Node.js and Express backend. Styling was implemented using CSS3, and JavaScript was used for client-side interactivity.

Key objectives included intuitive navigation, consistent branding, responsive layouts for all devices, and a solid foundation for upcoming sprint enhancements. By the end of Sprint 1, all main pages were functional and visually aligned with project goals.

UI Components

The following user interface components were created during Sprint 1:

  • Landing Page: Hero section with overlay text, prominent buttons linking to sprint documentation pages, and visually appealing backgrounds. Serves as the first impression for users.
  • Sign-Up Page: Users register through Google OAuth. No personal information is stored directly by the system. The page features a clear button for signing up with Google and provides a secure authentication experience.
  • Login Page: Authentication is handled via Google OAuth. Users simply click the login button to authenticate. No passwords or sensitive information are stored in the application. Client-side validations ensure smooth login experience.
  • Welcome Pages: Separate dashboards for Staff and Event Managers. Each dashboard displays personalized greetings and access to relevant functionalities. Designed for clarity, simplicity, and easy navigation.

Technologies Used

The frontend for Sprint 1 uses the following technologies:

  • EJS Templates: Modular HTML rendering integrated with the Node.js backend.
  • CSS3: Styling the pages with responsive design and modern aesthetics.
  • JavaScript: Handling interactivity such as button actions, navigation effects, and dynamic content updates.
  • Google OAuth: Secure third-party authentication without storing user credentials.

These technologies ensure modularity, maintainability, and readiness for future sprints.

Design Notes & Features

Key design principles applied in Sprint 1 include:

  • Responsive layouts for desktops, tablets, and mobile devices.
  • Consistent color palette and typography across pages.
  • Clear navigation and prominent action buttons for user interaction.
  • Modular design using EJS partials for headers, footers, and reusable components.
  • Accessible forms and buttons with hover effects and visual feedback.
  • Preparation for backend integration and dynamic content in upcoming sprints.

Overall, the frontend ensures a professional and cohesive user experience.
