Initial Design & Development Plan - Sprint 1

Project Architecture

For Sprint 1, the main architecture focuses on separating frontend and backend responsibilities, ensuring modularity and maintainability. We used an MVC-like approach with Node.js and Express handling backend routing, EJS templates for frontend rendering, and MongoDB as the database.

  • Frontend: EJS templates, CSS, and JavaScript for a responsive and dynamic user interface.
  • Backend: Node.js with Express handles authentication and routing.
  • Database: MongoDB stores minimal user-related data since authentication is handled via Google OAuth.
  • Authentication: Third-party Google OAuth for secure login of Staff and Event Managers.
  • Task Management: Notion was used to assign sprint tasks and track progress.
  • Architecture Diagram: Provides a visual overview of the system flow between frontend, backend, and database.

Notes & Rationale

This sprint focused on authentication and landing pages as the core foundation. Using Google OAuth ensures secure authentication without handling sensitive data directly. Task allocation via Notion allowed the team to work independently yet remain coordinated. The architecture diagram provides a clear understanding of the system flow, helping new team members onboard quickly.

By the end of Sprint 1, the authentication system was fully functional, the frontend landing pages were visually appealing and responsive, and the documentation site template was ready for teammates to populate further details.
